# 钱迹WorthTrace 手机端工作区

更新时间：2026-06-27

## 当前版本

- 手机 PWA 原型版本：0.3.4。
- 所属项目版本：2.6.2。
- 当前手机端通过电脑 Test App 的本地同步服务读取测试看板和提交草稿，不直接打开 SQLite 文件。
- 手机端本地草稿使用浏览器 IndexedDB：`worthtrace_mobile_v3`。
- 0.3.4 首次打开会清理旧 `worthtrace_mobile_v1` / `worthtrace_mobile_v2` 测试草稿，让手机端回到干净初始状态。

## 这个文件夹放什么

`mobile/` 专门放手机端相关内容。

包括：

- 手机端 mock
- 手机 PWA 原型
- 手机同步方案文档
- 离线记账和离线月末草稿设计
- 后续手机端接口说明

桌面正式 App 代码仍然在项目根目录的 `src/` 和 `src-tauri/`。

## 当前推荐方向

手机端第一阶段做：

1. 首页随身看板
2. 实时记账
3. 离线记账队列
4. 资产 Wallet 只读
5. 月底轻更新入口

第二阶段再做：

1. 离线资产更新草稿
2. 离线信用卡草稿
3. 同步冲突处理
4. 同步后电脑端校验并生成月报

## 文件夹说明

`mockups/`

手机端高保真方向稿。用于看视觉和交互方向，不接数据库。

`pwa/`

手机 PWA 原型。用于逐步落地手机页面、离线队列、同步提示。

`docs/`

手机端落地逻辑、同步边界、接口计划。

`assets/`

手机端会用到的品牌资产副本。这样手机 mock 和 PWA 不依赖外部路径。

## 当前边界

手机端 V1 可以离线做：

- 新增收入
- 新增支出
- 修改未同步记录
- 删除未同步记录
- 查看上一次同步过的摘要

手机端 V1 暂不离线做：

- 生成月报
- 关闭月份
- 最终 XIRR 计算
- 汇率锁定
- 正式更新财务健康看板
