# 手机端落地计划

更新时间：2026-06-12

## 目标

把手机端做成“日常记账 + 随身看板 + 月底轻更新”的入口。

核心变化：

以前：

`鲨鱼记账 → 导出文件 → 桌面导入 → 月底确认`

以后：

`手机实时记账 → 自动进入待同步队列 → 同 Wi-Fi 同步到电脑 → 月底只补资产和信用卡`

## 阶段 1：手机 UI 和 PWA 地基

要做：

1. 建立 `mobile/pwa/`
2. 做首页、记账、资产、月末入口 4 个页面
3. 做 PWA manifest
4. 做基础离线缓存 service worker
5. 页面上标明离线可用 / 需同步后完成

不做：

- 不接正式数据库
- 不改桌面 App 数据
- 不做云同步

## 阶段 2：离线记账队列

要做：

1. 手机本地保存记账记录
2. 每笔记录生成唯一 ID
3. 标记同步状态
4. 支持新增、修改、删除
5. 首页显示待同步数量

推荐字段：

- local_id
- server_id
- operation：create / update / delete
- sync_status：pending / synced / failed / conflict
- transaction_type：income / expense
- amount
- currency
- category
- account
- transaction_date
- note
- created_at
- updated_at

## 阶段 3：本地同步服务

要做：

1. 桌面端开放局域网同步接口
2. 手机 PWA 检测电脑服务是否在线
3. 同步待处理队列
4. 成功后写入电脑 SQLite
5. 返回同步结果

正式 / 测试 / demo 必须隔离。

## 阶段 4：接入桌面数据库

要做：

1. 手机新增收入支出写入桌面数据库
2. 桌面看板读取手机记账结果
3. 月底更新读取本月记账汇总
4. 如果本月已有手机记账数据，收支导入步骤变成“确认本月记账汇总”

## 阶段 5：月末轻更新

要做：

1. 手机显示月末轻更新入口
2. 手机在线时可填写资产和信用卡
3. 手机离线时先保存草稿
4. 同 Wi-Fi 后提示同步草稿
5. 电脑端校验后才允许生成月报

V1 不做离线月报生成。

## 离线状态标记

可以离线：

- 记收入
- 记支出
- 修改未同步记录
- 删除未同步记录
- 保存月末草稿，V1.1

需要同步后完成：

- 生成月报
- 关闭当月
- 汇率补齐
- XIRR 最终计算
- 看板正式更新
- 冲突处理

## 推荐第一版验收

1. 手机页面可打开
2. 可以新增一笔支出
3. 断网时记录进入待同步
4. 首页显示待同步数量
5. 手机恢复连接后弹出同步提示
6. 资产 Wallet 可以只读展示样例结构
7. 月底轻更新页明确哪些功能需要同步后完成

## 2026-06-13 UI 调整记录

### 顶部控制

手机端 4 个主页面右上角都保留同一组全局控制：

- 隐藏 / 显示金额
- 风格设置：香槟、鼠尾草、石墨
- 同步状态

三种风格参考桌面 App 当前同名主题。

### 首页本月提醒依据

当前 PWA 还是原型，没有接数据库。

因此首页“本月提醒”不再使用随手写的假提醒，而改成和桌面 App 首页逻辑一致的口径：

- 当前已经更新到哪个月份
- 已录入从哪个月份到哪个月份的数据
- 哪一天需要更新当月内容

后续接接口后，这三个值应该从桌面看板发布库读取。

### 记账分类依据

支出和收入不再显示“账户”字段。

分类支持：

- 默认分类
- 新增自定义分类

支出默认分类参考主流记账软件常见结构：

- 餐饮
- 交通
- 购物
- 居住
- 日用
- 医疗
- 娱乐
- 旅行
- 人情
- 学习
- 运动
- 宠物
- 保险
- 税费
- 其他支出

收入默认分类：

- 工资
- 奖金
- 副业
- 投资收益
- 分红
- 利息
- 报销
- 退款
- 红包
- 租金
- 其他收入

### 主模块调整

底部导航从：

`首页 / 记账 / 资产 / 月末`

调整为：

`首页 / 记账 / 收支 / 资产`

原因：

- 月末轻更新不适合作为单独底部模块
- 手机端应更接近桌面健康看板的移动拆分
- 月末更新作为首页提醒和资产页里的入口存在
